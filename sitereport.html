Display And Design

For my website I decided I wanted the design to be fun but simplistic in order to make it easier to navigate and keep the pages relevant. I chose a fox gif for the background as its my favourite animal and coincidently my last name which makes the website seem more personal. The navigation bar at the top of the screen displays which page is currently open and displays all the pages my site includes, for mobile the navigation bar switches to a simple burger menu which also displays the different pages. I decided to make the layout similar to most websites on the internet as most visitors tend to have an understanding of where the navigation menu usually sits. I chose colours for my pages which contrast nicely so the text displayed was easily readable. The use of boxes to contain the text produced even more of a contrast rather than sitting on the white background of the page and made the website look more professional. After experimenting with many different text fonts I decided the one that worked best was easily readable at different sizes and still looked professional while adding style.

Difficulties and Fixes

I overall found learning how to use Visual Studio Code easy to understand and use, as well as understanding the different tags used in the HTML and how to use CSS files to style the pages. Producing the content and styling for the website was simple but it was laying out the page for the content I sometimes experienced difficulties. There was a lot of fine tuning involved in order for the website to look aesthetically pleasing on both computer and mobile version but I found using different sections in the body of the HTML made laying the content on the grid much simpler rather than it all being cluttered in one section. The pages used a lot of the same HTML and CSS so it was very easy implementing items such as the navigation bar and burger menu which stayed fairly similar on each page. The use of GitHub in the design process of the website at first was difficult to understand but through lectures and tutorials, my understanding was quickly established and I was able to use GitHub easily and effectively to produce multiple branches for each of my pages and upload meaningful comments when designing and editing my code. Using GitHub is beneficial when testing and making changes to the website as it meant I could edit different aspects of the page without interrupting the original website.
